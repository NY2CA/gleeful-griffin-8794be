# Wave 16.1 — Editorial Weekly Reads with Autofeed Default

**Shipped:** May 8, 2026
**Scope:** Real, fresh weekly reads serving both Mastery Live and Self-Study dashboards. **Default flow is autofeed** — a Tuesday cron uses Claude with web search to publish 3 fresh multifamily articles every week with no human action. Lou and Diva can refresh / edit / swap / delete any time, any day.

## Architecture

**Editorial source of truth lives on Mastery.** Self-Study fetches via Mastery's public CORS-open endpoint, so one Publish (manual or autofeed) updates both dashboards.

**Three layers:**

1. **Tuesday autofeed** (`scheduled-weekly-reads.ts`) — runs every Tuesday at 8am ET. Calls Claude Haiku with the `web_search_20250305` tool, asks for 3 fresh multifamily articles from the past 7 days, parses the JSON response, auto-publishes to the `weeklyReads` blob with `autoGenerated: true`. Free of human input. Cost: ~$0.01–0.02 per Tuesday run.
2. **Manual override** (`/admin/weekly-reads` page) — Lou and Diva open the form any day, edit any row, delete any row, add up to 5 total, click Publish. Manual publishes set `autoGenerated: false` and stay live until the next Tuesday autofeed cycle replaces them.
3. **Refresh-on-demand** ("Refresh from autofeed" button on the admin form) — admin clicks → Claude pulls 3 fresh suggestions → form rows pre-populate. Admin reviews, edits, swaps individual articles, then clicks Publish. Use this any day of the week to pull fresh content without waiting for Tuesday.

**Defaults:** 3 articles. Admin can extend to **5 max** via "Add another" on the form. 5 is the hard cap (server-side validation rejects more).

## Editorial workflow examples

**Tuesday morning, default case:**
- 8am ET: cron fires, autofeed runs, 3 articles auto-publish, both dashboards refresh within 60 seconds. No human action needed.

**Tuesday morning, manual override:**
- 9am ET: Lou opens `/admin/weekly-reads`, sees the autofeed published 3 hours earlier
- Reviews articles. One is good, one is okay, one isn't relevant.
- Edits the okay one's "why this matters" line. Swaps the third one with a Substack post he wrote.
- Clicks Publish. The new mix replaces the autofeed.

**Mid-week swap:**
- Thursday: Diva sees a great NMHC report. Opens `/admin/weekly-reads`.
- Adds a 4th row with the NMHC article, fills in the fields.
- Clicks Publish. Dashboard updates within 60 seconds.

**Refresh from autofeed any day:**
- Friday: Lou wants to feature recent debt market news but doesn't have specific links in mind.
- Opens `/admin/weekly-reads`, clicks **Refresh from autofeed**.
- Claude returns 3 fresh suggestions in ~5 seconds.
- Lou reviews, deletes one he doesn't like, adds his own custom 3rd article.
- Clicks Publish.

## What you can publish

Pure manual editorial control on the override path. Each row accepts:

- **Source** (any string · "CBRE", "Multi-Housing News", "NMHC", "Rescia Properties", "Lou's notes" — whatever)
- **Date label** (e.g. "May 5" · optional)
- **Title** (article headline)
- **Link** (any URL — third-party article, your own Substack, a Rescia memo, anything)
- **Why this matters** (one-line framing for members · 280-char cap · optional)

Empty rows dropped on Publish. Per-row Remove button to delete. Hard cap: 5 articles.

## Cost note

The autofeed runs once per week → ~$0.01–0.02 per run → ~$1/year of Anthropic API. The Refresh button costs the same per click — call it whenever you want fresh suggestions. Negligible.

## What if autofeed fails?

Silent fail — log goes to Function logs. The previous published blob stays live (members keep seeing the prior week's articles). Admin can manually publish anytime to recover. No member-visible breakage.
